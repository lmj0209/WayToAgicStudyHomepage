# 产品需求文档：个人学术主页 - V7.0 - 缩小侧边栏与内容间距

## 1. 综述 (Overview)

### 1.1 项目背景

基于 PRD 6.0 的实施反馈，用户发现侧边栏与主内容区之间的间距仍然过大（gap-8 = 2rem = 32px）。本版本专注于**进一步缩小侧边栏与内容区的间距**，使其更接近参考设计 `reference/2.png` 的视觉效果。

### 1.2 核心问题

| 问题 | 当前状态 | 期望状态 |
|------|----------|----------|
| 侧边栏与内容区间距 | gap-8 (32px) | **缩小至 gap-4 或 gap-6** |
| 视觉效果 | 间距过大，内容分散 | 间距适中，内容紧凑 |

### 1.3 参考设计分析

根据 `reference/2.png` 的视觉分析：
- **侧边栏与内容间距**：参考设计的侧边栏（右侧）与主内容区之间的间距较小，约 16-24px
- **视觉紧凑性**：整体布局紧凑，不会有过大留白

---

## 2. 用户故事详述 (User Stories)

---

### **US-19: 缩小侧边栏与主内容区的间距**

*   **价值陈述 (Value Statement)**:
    *   **作为** 访问者
    *   **我希望** 侧边栏与内容区之间的间距更小
    *   **以便于** 页面布局更紧凑，视觉更协调

*   **业务规则与逻辑**:
    1.  **前置条件**: PRD 6.0 已完成实施
    2.  **当前值**: `gap-8` (32px)
    3.  **目标值**:
        - 方案 A: `gap-6` (24px) - 适度缩小
        - 方案 B: `gap-4` (16px) - 明显缩小
        - 方案 C: `gap-3` (12px) - 最小间距

*   **验收标准**:
    *   **场景1: 间距大小**
        *   **GIVEN** 用户在桌面端浏览
        *   **WHEN** 查看侧边栏与内容区
        *   **THEN** 间距适中，约 16-24px
    *   **场景2: 响应式保持**
        *   **GIVEN** 调整间距
        *   **WHEN** 在移动端查看
        *   **THEN** 移动端布局保持正常

*   **需要修改的文件**:
    ```
    app/page.tsx              # 修改 gap 值
    app/academic/page.tsx      # 修改 gap 值
    ```

---

## 3. 具体修改方案

### 3.1 间距方案对比

| 方案 | Tailwind 类 | 像素值 | 视觉效果 |
|------|------------|--------|----------|
| A | gap-6 | 24px | 适度缩小，平衡 |
| B | gap-4 | 16px | 明显缩小，紧凑 |
| C | gap-3 | 12px | 最小间距，最紧凑 |

### 3.2 推荐方案：gap-4

基于参考设计的视觉分析，推荐使用 `gap-4`（16px），既保持适当的间距，又不会过于分散。

### 3.3 修改位置

**app/page.tsx 和 app/academic/page.tsx**:

```tsx
// 当前
<div className="flex flex-col lg:flex-row gap-8">

// 修改为
<div className="flex flex-col lg:flex-row gap-4">
```

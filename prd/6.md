# 产品需求文档：个人学术主页 - V6.0 - 侧边栏位置与AI编程页介绍

## 1. 综述 (Overview)

### 1.1 项目背景

基于 PRD 5.0 的实施反馈，用户提出两个新的调整需求：
1. **侧边栏位置调整**：侧边栏与主内容区距离过大，需要将侧边栏往右移动，使其位于顶部 "Mengjin Li" 名字的下方（紧贴导航栏右侧）
2. **AI 编程页面增加自我介绍**：在 AI 编程页面添加类似学术页面 About Me 的自我介绍模块

### 1.2 核心问题

| 问题 | 当前状态 | 期望状态 |
|------|----------|----------|
| 侧边栏位置 | 左侧独立栏，内容区有 lg:ml-[300px] | 侧边栏移至内容区顶部右侧 |
| 侧边栏与内容区间距 | 300px（侧边栏宽度） | 内容正常宽度，侧边栏在右侧 |
| AI 编程页面介绍 | 无 | 添加 About Me 风格的自我介绍 |

### 1.3 参考设计分析

根据 `reference/2.png` 的视觉分析：
- **侧边栏位置**：参考设计的侧边栏实际是在主内容区的**右上角**，而不是左侧独立栏
- **布局结构**：导航栏下方，主内容区包含侧边栏信息（右侧）和主要内容（左侧/中间）
- **响应式设计**：移动端侧边栏折叠到顶部，桌面端显示在右侧

## 2. 用户故事详述 (User Stories)

---

### **US-17: 将侧边栏移至内容区右上角**

*   **价值陈述 (Value Statement)**:
    *   **作为** 访问者
    *   **我希望** 侧边栏位于内容区右上角，而不是左侧独立栏
    *   **以便于** 内容区宽度更充分利用，侧边栏信息更靠近内容

*   **业务规则与逻辑**:
    1.  **前置条件**: PRD 5.0 已完成实施
    2.  **当前布局结构**:
        ```
        <Header />
        <div className="flex">
          <Sidebar />          <!-- 固定在左侧 -->
          <main className="lg:ml-[300px]">  <!-- 主内容区左边距 300px -->
            <div className="max-w-3xl">内容</div>
          </main>
        </div>
        ```
    3.  **目标布局结构**:
        ```
        <Header />
        <main className="max-w-5xl mx-auto px-4">
          <div className="flex gap-8">
            <div className="flex-1">内容</div>
            <Sidebar />        <!-- 在右侧 -->
          </div>
        </main>
        ```
    4.  **实现要点**:
        - 移除侧边栏的 fixed 定位
        - 侧边栏改为内容区内的 flex 布局
        - 移除主内容区的左边距
        - 侧边栏在桌面端显示在右侧，移动端保持折叠在顶部

*   **验收标准**:
    *   **场景1: 桌面端布局**
        *   **GIVEN** 用户在桌面端浏览
        *   **WHEN** 查看页面布局
        *   **THEN** 侧边栏显示在内容区右上角，主内容区在左侧/中间
    *   **场景2: 内容区宽度**
        *   **GIVEN** 侧边栏移至右侧
        *   **WHEN** 查看主内容区
        *   **THEN** 内容区宽度不再受侧边栏影响，可以充分利用空间
    *   **场景3: 移动端布局**
        *   **GIVEN** 用户在移动端查看
        *   **WHEN** 浏览页面
        *   **THEN** 侧边栏默认折叠为顶部个人信息条，点击可展开

*   **需要修改的文件**:
    ```
    app/page.tsx              # 调整布局结构
    app/academic/page.tsx      # 调整布局结构
    components/Sidebar.tsx     # 修改定位方式（移除 fixed，改为 flex）
    ```

### **US-18: AI 编程页面增加自我介绍模块**

*   **价值陈述 (Value Statement)**:
    *   **作为** 访问者
    *   **我希望** 在 AI 编程页面也能看到个人简介
    *   **以便于** 了解博主背景，增强信任感

*   **业务规则与逻辑**:
    1.  **前置条件**: PRD 5.0 已完成实施，AI 编程页面为首页
    2.  **当前 AI 编程页面结构**:
        - 🚀 快速学习推荐
        - 📚 学习笔记
        - 🤖 工具推荐
    3.  **目标 AI 编程页面结构**:
        - 👋 About Me（新增）
        - 🚀 快速学习推荐
        - 📚 学习笔记
        - 🤖 工具推荐
    4.  **自我介绍内容要点**:
        - 个人简介：博士生，研究方向
        - AI 编程学习经历：何时开始，使用过哪些工具
        - 学习心得：对 AI 辅助编程的看法
        - 联系方式：鼓励交流

*   **验收标准**:
    *   **场景1: 模块显示**
        *   **GIVEN** 用户访问 AI 编程页面
        *   **WHEN** 查看页面内容
        *   **THEN** 最上方显示 About Me 模块
    *   **场景2: 内容相关性**
        *   **GIVEN** About Me 模块已显示
        *   **WHEN** 阅读内容
        *   **THEN** 内容聚焦于 AI 编程学习相关，与学术页面的研究导向区分
    *   **场景3: 样式一致**
        *   **GIVEN** 查看两个页面的 About Me 模块
        *   **WHEN** 对比样式
        *   **THEN** 样式保持一致（标题样式、间距、字体大小）

*   **需要修改的文件**:
    ```
    components/AICodingPage.tsx  # 添加 AboutMe 模块
    components/AIAboutMe.tsx     # 新建 AI 编程页面的 About Me 组件
    ```

## 3. 具体修改方案

### 3.1 侧边栏位置调整

**当前布局**:
```tsx
<main className="flex-1 lg:ml-[300px] min-h-screen">
  <div className="max-w-3xl mx-auto px-4 py-8 md:px-6 lg:px-8">
    <Content />
  </div>
</main>
```

**修改后布局**:
```tsx
<main className="w-full px-4 py-8 md:px-6 lg:px-8">
  <div className="max-w-6xl mx-auto">
    <div className="flex flex-col lg:flex-row gap-8">
      <div className="flex-1 min-w-0">
        <Content />
      </div>
      <div className="lg:w-[280px] lg:flex-shrink-0">
        <Sidebar />
      </div>
    </div>
  </div>
</main>
```

**Sidebar 组件调整**:
- 移除 `fixed` 定位
- 移除 `w-[300px]` 固定宽度
- 改为由父容器控制宽度

### 3.2 AI 编程页面 About Me 内容

**标题**: 👋 About Me

**内容示例**:
```markdown
我是一名专注于 AI 辅助编程的学习者和实践者。作为一名计算机学院的博士生，我深感 AI 工具对提升编程效率的巨大帮助。

**我的 AI 编程之旅**

从 2023 年开始，我深入使用了多种 AI 编程工具，包括 Claude Code、GitHub Copilot、Cursor 等。在日常科研和开发工作中，这些工具帮助我：

- 快速实现原型想法
- 理解和维护遗留代码
- 学习新的编程语言和框架
- 自动化重复性任务

**学习心得**

我认为 AI 编程不是要取代程序员，而是要成为程序员的"超级助手"。关键是学会如何与 AI 有效沟通，清晰地表达需求。

如果你也对 AI 编程感兴趣，欢迎通过侧边栏的联系方式和我和交流！
```
